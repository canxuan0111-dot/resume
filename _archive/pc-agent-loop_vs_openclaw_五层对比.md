# pc-agent-loop vs OpenClaw：基于 5 层框架的对比

> 目标：用统一框架评估两类 Agent 产品路线，帮助快速理解差异与取舍。

## 一、对比对象

- **pc-agent-loop**（`lsdefine/pc-agent-loop`）
  - 定位偏个人自动化代理
  - 强调任务执行后沉淀 SOP，能力随使用增长
- **OpenClaw**
  - 定位偏平台化 Agent 运行系统
  - 强调工具化、会话化、可治理与跨渠道编排

---

## 二、5 层设计框架

1. 产品目标层
2. 交互与任务层
3. Agent 运行机制层
4. 系统架构层
5. 治理与运营层

---

## 三、逐层对比

## 1) 产品目标层

### pc-agent-loop
- 核心目标：让 Agent 直接“替你动手”完成真实设备任务（PC/手机）
- 价值主张：少量初始能力 + 持续任务沉淀，越用越强
- 目标用户：个人用户、自动化爱好者、偏实验/实战场景

### OpenClaw
- 核心目标：让 Agent 在多工具、多会话、多渠道下稳定运行
- 价值主张：结构化能力治理 + 可扩展 + 可运营
- 目标用户：个人到团队，尤其需要长期可控运行的场景

### 关键差异
- **pc-agent-loop 更偏“个人效率爆发”**
- **OpenClaw 更偏“平台化长期运营”**

---

## 2) 交互与任务层

### pc-agent-loop
- 交互偏目标驱动：给任务后，系统自主探索执行路径
- 特点：首次慢、后续快（依赖 SOP 复用）
- 风险：任务描述不清时，执行路径可能偏离预期

### OpenClaw
- 交互偏结构化：会话、工具调用、子代理、跨会话消息都有明确机制
- 特点：复杂任务可拆分、可追踪
- 风险：前期理解工具体系有学习成本

### 关键差异
- **pc-agent-loop：更像“交代结果，先做出来”**
- **OpenClaw：更像“在可控流程里把结果做出来”**

---

## 3) Agent 运行机制层

### pc-agent-loop
- 机制核心：任务执行 → 经验抽取 → SOP 沉淀 → 下次复用
- 偏向“经验生长型智能体”
- 强项：对重复、贴近个人习惯的任务提效明显

### OpenClaw
- 机制核心：工具编排 + 技能（Skills）+ 会话治理 + 调度
- 偏向“模块化能力系统”
- 强项：复杂流程可组合，可分工（sub-agents），可控性强

### 关键差异
- **SOP（隐式经验积累）** vs **Skills（显式能力模块）**
- 一个偏“从做中学”，一个偏“先定义边界再稳定复用”

---

## 4) 系统架构层

### pc-agent-loop
- 架构倾向：轻量、单体感更强，围绕本机实操能力构建
- 优势：上手快、路径直接
- 挑战：多人协作、复杂权限治理、统一观测能力可能较弱

### OpenClaw
- 架构倾向：工具接口标准化、会话系统化、任务调度化
- 优势：可扩展、可编排、可接入多通道
- 挑战：系统层复杂度更高，需要更规范的配置与维护

### 关键差异
- **pc-agent-loop：轻量与直达**
- **OpenClaw：系统化与可扩展**

---

## 5) 治理与运营层

### pc-agent-loop
- 治理重点：SOP 质量控制、历史流程清理、失效流程更新
- 典型问题：经验积累快，但也可能积累“坏流程”

### OpenClaw
- 治理重点：权限边界、审计日志、会话隔离、成本与模型策略
- 典型优势：治理能力更容易做成组织级规范

### 关键差异
- **pc-agent-loop：靠持续使用打磨个体智能体**
- **OpenClaw：靠治理体系保证长期稳定与可控**

---

## 四、总结（不做选型建议，仅给定位）

- 如果把两者看成同一赛道：
  - **pc-agent-loop** 更像“高自主、经验驱动的个人执行型 Agent”
  - **OpenClaw** 更像“可治理、可扩展的 Agent 平台基础设施”

- 本质不是“谁替代谁”，而是产品哲学不同：
  - 一个优先解决“今天把事做成”
  - 一个优先解决“长期稳定地持续做成”

---

## 五、可复用评估模板（后续可套其他产品）

对任意 Agent 产品，按以下 5 问快速评估：

1. **产品目标层**：它最想优化的是“速度”还是“可控性”？
2. **交互任务层**：任务是自由描述优先，还是结构化流程优先？
3. **运行机制层**：能力来自运行后学习，还是预定义工具模块？
4. **系统架构层**：是单体快用，还是平台化可扩展？
5. **治理运营层**：审计、权限、成本、版本如何落地？

> 以上框架可用于你后续对比 Claude Code / Manus / AutoGen / LangGraph 等方案。
